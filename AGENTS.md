# AGENTS.md

## Scope
- These instructions apply to the entire repository unless a deeper `AGENTS.md` overrides them.

## Project Overview
- This repository is the frontend for `ifsc.stream`, built as a static Astro site.
- Astro pages/components live in `src/` and compile to static output in `dist/`.
- Browser JavaScript is authored in `src/js/app/` and bundled into `public/js/app.js` via `bin/build-js-bundle.mjs`.
- `public/` contains static assets consumed at build time.
- PHP scripts in `bin/` generate SEO/data artifacts and cache athlete photos.
- Playwright integration tests validate frontend behavior against the built site.

## Routing And Rendering
- `/` is generated from `src/pages/index.astro`.
- `/season/:season` pages are prerendered from `src/pages/season/[season]/index.astro`.
- `/start-list-modals/:season/:eventId.html` fragments are prerendered from `src/pages/start-list-modals/[season]/[eventId].html.js`.
- Season/event JSON from `events/` is loaded during build (not fetched via runtime AJAX for core page data).

## Key Paths
- `src/pages/index.astro` is the main site page source.
- `src/pages/season/[season]/index.astro` defines season page routes.
- `src/components/` contains Astro UI components for the calendar experience.
- `src/js/app/` contains source JavaScript modules for frontend behavior.
- `public/js/app.js` is generated from `src/js/app/` by `bin/build-js-bundle.mjs`.
- `public/css/` contains hand-maintained stylesheets.
- `events/` contains season event data JSON files.
- `public/structured-data/` contains generated JSON-LD artifacts.
- `public/sitemap.xml` is generated by `bin/build-sitemap`.
- `dist/` is generated static site output from Astro.
- `tests/integration/` contains Playwright integration tests.
- `Makefile` provides the primary test entrypoints (`test`, `test-local`, `test-docker`).

## Ignore files
- `.gitignore` is used to ignore generated outputs and local-only files.
- `tests/integration/reports/` contains Playwright reports.
- `public/js/app.js` is generated from `src/js/app/` by `bin/build-js-bundle.mjs`.

## Working Rules
- Preserve the Astro + static-output architecture and existing frontend behavior.
- Keep changes surgical and consistent with the existing style of the touched file.
- Prefer editing Astro pages/components in `src/` instead of generated `dist/` output.
- Prefer editing JS source files in `src/js/app/`; regenerate `public/js/app.js` after JS source changes.
- Do not introduce new frameworks, build systems, or TypeScript unless explicitly requested.
- Treat generated directories/files (`dist/`, `.astro/`, `vendor/`, `node_modules/`, `public/js/app.js`) as generated output; do not hand-edit them.
- Avoid large formatting-only diffs in Astro templates, CSS, JS, and JSON data files.
- Maintain compatibility with the existing Bootstrap and jQuery-based frontend patterns already in the repo.
- Keep Playwright tests stable by preserving selectors, modal behavior, filter behavior, and season navigation unless intentionally changing UX.

## Data And Assets
- Event JSON files in `events/` are source-like data for the site; update them carefully and keep schema compatibility with existing JS.
- Files under `public/structured-data/` and `public/sitemap.xml` are generated artifacts; regenerate with the corresponding `bin/` scripts when source data changes.
- Athlete photos under `public/img/athletes/` are generated/refreshed via `bin/cache-athlete-photos`.
- Respect existing ignore rules for generated outputs such as `dist/`, `.astro/`, `vendor/`, `node_modules/`, Playwright reports, JS bundles, and ignored current-year artifacts.

## Commands
- Install PHP dependencies with `composer install`.
- Install frontend test dependencies with `npm install --no-package-lock`.
- Run Astro dev server with `npm run dev`.
- Build the site with `npm run build`.
- Preview the built site with `npm run preview`.
- Rebuild browser JS bundle only with `npm run build:js` (or `npm run build:js:min`).
- Run a local static server for `dist/` with `make serve` (requires `caddy`).
- Run integration tests (auto-picks Docker if available): `make test`.
- Run integration tests on host machine: `make test-local`.
- Run integration tests in Playwright Docker image: `make test-docker`.
- Build structured data files: `php bin/build-structured-data [events-dir] [output-dir] [site-url]`.
- Build sitemap: `php bin/build-sitemap [events-dir] [output-file] [site-url]`.
- Cache and resize athlete photos: `php bin/cache-athlete-photos [events-dir] [images-dir] [image-size]`.

## Validation
- Playwright integration tests are available in `tests/integration/`.
- For frontend changes, run `npm run build` and then `make test` when possible.
- Do lightweight manual verification with `npm run dev` or `npm run preview` for key flows (`/`, `/season/:season`, modals/filters/start-list modal loads).
- For script changes, run only the narrowest relevant script command and verify generated artifacts without touching unrelated outputs.

## Ignore Files
- Keep `.gitignore` aligned with generated outputs and local-only files.
- If a change introduces a new generated artifact or dependency directory, update `.gitignore` in the same task.
