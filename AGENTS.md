# AGENTS.md

## Scope
- These instructions apply to the entire repository unless a deeper `AGENTS.md` overrides them.

## Project Overview
- This repository is a mostly static frontend for `ifsc.stream`.
- The deployed site lives in `public/`.
- There is no frontend build step in this repo.
- PHP is used for small maintenance scripts in `bin/`.
- Composer manages the PHP dependency used by those scripts.

## Key Paths
- `public/index.html` is the main site entrypoint.
- `public/css/` contains hand-maintained stylesheets.
- `public/js/` contains hand-maintained browser JavaScript.
- `public/events/` contains season event data JSON files.
- `public/feed/` contains RSS output artifacts.
- `public/img/` contains committed static images plus generated poster/news assets.
- `bin/download-posters` downloads and resizes event posters.
- `bin/download-news-images` downloads and resizes news images.

## Working Rules
- Preserve the current no-build architecture; prefer editing the existing HTML, CSS, and vanilla JavaScript directly.
- Keep changes surgical and consistent with the existing style of the touched file.
- Do not introduce frameworks, bundlers, transpilers, or TypeScript unless the user explicitly asks for them.
- Treat `vendor/` as generated dependency output; do not hand-edit it.
- Avoid large formatting-only diffs in `public/index.html`, CSS, or JS files.
- Maintain compatibility with the existing Bootstrap and jQuery-based frontend patterns already in the repo.

## Data And Assets
- Event JSON files in `public/events/` are source-like data for the site; update them carefully and keep schema compatibility with existing JS.
- Files under `public/img/news/` and `public/img/posters/` may be generated by helper scripts; do not delete committed assets unless the user asks.
- Respect existing ignore rules for generated outputs such as `vendor/`, RSS output, and current-year event data artifacts.

## Commands
- Install PHP dependencies with `composer install`.
- Run a local static preview with `php -S 127.0.0.1:8000 -t public`.
- Refresh posters with `php bin/download-posters <season> <events-json-path>`.
- Refresh news images with `php bin/download-news-images <rss-xml-path>`.

## Validation
- There is no formal test suite configured in this repo.
- For frontend changes, prefer lightweight manual verification by serving `public/` locally.
- For script changes, validate with the narrowest relevant command and avoid touching unrelated generated assets.

## Ignore Files
- Keep `.gitignore` aligned with generated outputs and local-only files.
- If a change introduces a new generated artifact or dependency directory, update `.gitignore` in the same task.
