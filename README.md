# World Climbing (IFSC) Calendar and Streams Frontend
![Deploy static content to Pages](https://github.com/sportclimbing/web/actions/workflows/static-deploy.yml/badge.svg)

Frontend for [ifsc.stream](https://ifsc.stream/), built as a static Astro site.

It renders World Climbing season/event data generated by [sportclimbing/ifsc-calendar](https://github.com/sportclimbing/ifsc-calendar) and deploys static output to GitHub Pages.

## Current status

- Astro pages/components live in `src/`, static output is generated in `dist/`.
- Browser app code lives in `src/js/app/` and is bundled into `public/js/app.js`.
- Season pages are prerendered at `/season/:season`.
- Event pages are prerendered at `/season/:season/event/:eventId`.
- Start-list modals are prerendered at `/start-list-modals/:season/:eventId.html`.
- Desktop/mobile styles are split at `screen and (min-width: 801px)` and `screen and (max-width: 800px)`.

## Local development

- Install dependencies: `npm install --no-package-lock`
- Start dev server: `npm run dev`
- Build site: `npm run build`
- Preview build: `npm run preview`
- Rebuild JS only: `npm run build:js` (or `npm run build:js:min`)

## Data and generated artifacts

- Season source data: `events/events_YYYY.json`
- Structured data build: `php bin/build-structured-data`
- Sitemap build: `php bin/build-sitemap`
- Athlete photo cache/resize: `php bin/cache-athlete-photos`

## Integration tests

Playwright integration tests live in [`tests/integration`](tests/integration).

- `make test` auto-selects Docker when available (recommended).
- `make test-local` runs on host machine.
- `make test-docker` forces Playwright Docker image.

## CI and deploy

Workflow: [`.github/workflows/static-deploy.yml`](.github/workflows/static-deploy.yml)

- Triggered on `main` pushes, hourly schedule, and manual dispatch.
- Test job runs Playwright integration tests.
- Deploy job rebuilds data artifacts, builds Astro output, and deploys `dist/` to GitHub Pages.
